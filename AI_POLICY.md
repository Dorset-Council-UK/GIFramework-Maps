# AI Use Policy
Version 1.0
Last updated: 2026-04-24

## Introduction

This document outlines the policy for the use of Artificial Intelligence (AI) within our organisation. The purpose of this policy is to ensure that AI technologies are used responsibly, ethically, and in compliance with applicable laws and regulations.

This policy outlines a web development specific guide to the use of AI which is in line with the Dorset Council's broader AI policy which can be found at [Dorset Council AI Policy](https://www.dorsetcouncil.gov.uk/w/policy-for-the-use-of-ai-automation-algorithmic-data-processing-in-dorset-council).

For clarity on specific points the full policy should be used. 

This application is released under an open-source license, however, we still need to protect the integrity of the project and ensure that any use of AI-generated content is used responsibly and that sources are properly cited.

## Guidelines for AI Use
1. Contributors are responsible for every line of code they submit. You must:
   - Apply the `AI` label to your PR if any feature, function, fix, or significant portion of code was generated by AI. The label is not required if you only used AI for advice during development.
   - Certify that you have read and understood every line of code submitted.
   - Take appropriate measures to ensure that:
	 - You know where the code has come from and properly cite all sources in the pull request description.
	 - Confirm that you have the right to use any AI-generated content.
2. Large rewrites done wholly by AI are likely to be rejected without adequate explanation.
3. GitHub Copilot code review is enforced on all pull requests to the main branch on this project and can help identify minor issues. However, the developer remains responsible for validating any suggestions made.
4. We recommend carefully crafting your prompts to direct AI agents towards known sources, such as official documentation. This makes it easier to cite references and verify the accuracy of generated content. 

All contributions must be submitted as a pull request. Every line of code must be understood by a `core developer` on this project, and you may be asked to explain AI-generated code.

See [CONTRIBUTING.md](./CONTRIBUTING.md) for more information on becoming a core developer.

## Policy development

The use of AI is constantly evolving and this policy will be updated regularly as our understanding grows. Please check this document periodically for changes.

Maintaining a human in the loop is essential to keeping this codebase stable and ensuring issues can be resolved quickly.

This policy should be read alongside the project's [CONTRIBUTING.md](./CONTRIBUTING.md), [DEVELOPING.md](./DEVELOPING.md), and [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) documents.
